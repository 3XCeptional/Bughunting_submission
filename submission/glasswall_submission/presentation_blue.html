Vulnerability — The Guard Who Checks No IDs

🔐 OWASP API Security #1

BOLA — Broken Object Level Authorization

Guard checks your badge at the door — you're in. But every room inside is left unlocked. No ownership check. No role boundary.

✓ Server DOES Check

✓ Valid bearer token
✓ Token not expired
✓ User exists in system
✓ Hardcoded default → 403

✗ Server NEVER Checks

✗ Does user own this resource?
✗ Did user create this resource?
✗ Does role allow write/delete?
✗ OID in JWT = resource owner?

Affected: GET/PUT/DELETE /api/v1/policies/{name} & GET/PUT/DELETE /api/v1/profiles/{name}

CWE-639 CWE-284 OWASP API 2023 — A1

The Fix — One Check. Consistently Applied.
